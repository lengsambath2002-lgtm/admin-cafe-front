# Mobile → backend Dev (mobile-api) Change Requests

Log for change requests filed by the Mobile role when the backend mobile-api
needs a new or changed endpoint, field, error contract, or auth behaviour.
Backend Dev owns mobile-api (repo: D:/admin-cafe), triages each request, and
either implements it or declines. Mobile never edits the backend directly.

Format per entry:

    ### CR-MA-### — <summary>
    - Requested by: <name> (Mobile, iOS|Android), <date>
    - Endpoint/area: <e.g. POST /api/khqr/check, GET /api/me>
    - Status: open | in-progress | done | declined
    - Context: <what the app needs and why>
    - Ask: <the concrete contract change — fields, types, status codes, errors>
    - Resolution: <filled when done — link to the spec section / MR>
