# Dev → PO Change Requests

Log for requests filed by Dev when a brief is missing, wrong, or ambiguous.
PO triages each and either clarifies the brief or declines. Dev never edits
docs/po/ directly.

Format per entry:

    ### CR-PO-### — <summary>
    - Requested by: <name> (Dev), <date>
    - Brief: <link to docs/po/briefs/...>
    - Status: open | in-progress | done | declined
    - Context: <what's unclear / missing>
    - Ask: <the concrete clarification or change needed>
    - Resolution: <filled when done — what PO decided / link>
